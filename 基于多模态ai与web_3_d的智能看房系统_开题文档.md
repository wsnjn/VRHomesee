# 一、选题的背景与意义

## 1. 研究背景

当前，主流线上租赁平台的数字化展示仍普遍停留在“二维图片列表”形式，缺乏对房源空间属性与居住环境的深度还原与理解。用户在浏览房源时，往往只能依赖静态图片与文字描述，难以获得身临其境的空间感受，也无法围绕布局、采光、装修细节等视觉要素进行即时交互与深入询问。这种信息表达方式与用户真实看房需求之间存在明显断层，导致线上决策效率低下，线下重复看房成本高企。

在此背景下，本研究设计并实现了一套融合 Web 3D 沉浸式展示与多模态大模型智能对话的线上看房系统。系统不仅通过全景与三维技术对房源环境进行真实还原，还引入视觉—语言融合理解模型，使 AI 助手具备“看懂”房源场景并基于视觉内容进行问答与推理的能力，从而突破传统基于关键词检索或纯文本客服的交互局限，实现真正意义上的“所见即所问”。

## 2. 理论意义

本研究从理论层面探索了多模态大模型在垂直领域（房地产场景）中的落地应用范式，验证了“视觉感知—自然语言理解—交互反馈”闭环机制在真实业务系统中的可行性与有效性。研究成果为情境感知计算、多模态人机交互以及智能导览系统提供了新的实践案例。

此外，本文构建并验证了基于检索增强生成（RAG）的智能推荐框架，突破了传统协同过滤或标签匹配算法的局限，展示了基于深度语义理解与视觉内容分析实现个性化推荐的可行路径，为相关领域后续研究提供了理论与方法参考。

## 3. 实践意义

在实践层面，本研究具有较强的工程与应用价值。首先，在交互范式上实现了“Web 端沉浸式全景漫游”与“小程序端轻量化交互体验”的双端协同，验证了在移动端环境下通过混合架构集成 3D 渲染与 AI 服务的可行性，为复杂 Web 应用的小程序化开发提供了实践经验。

其次，在技术应用层面，将大模型能力从传统文本对话拓展至三维视觉环境理解与推理，实现了“即看即问”的沉浸式导览体验，显著提升了用户的信息获取效率与使用满意度。

最后，在行业层面，本研究为房产租赁行业的数字化升级提供了一种从“静态展示”迈向“理解与交互”的新方案，在提升用户决策效率与看房体验的同时，具备明确的商业落地与推广价值。

# 二、课题研究的基本内容与拟解决的主要问题

## 1. 课题研究的基本内容

本系统旨在构建一个“可看、可问、可交互”的下一代智能看房平台。整体架构由双端交互层、多模态 AI 中台与 3D 数字孪生层协同构成。

### （1）双端全景看房交互层

Web 端采用 Vue3 与 Three.js 技术栈，实现高性能的全景漫游功能，支持自由视角旋转、缩放及热点导航，为用户提供桌面级沉浸式看房体验。

小程序端针对移动设备性能约束，采用原生小程序框架与 Web-View 组件相结合的混合架构。通过 Web-View 承载 Web 端 3D 核心模块，并利用 JSBridge 实现原生组件与 3D 页面之间的双向通信，在保障沉浸体验的同时兼顾性能与兼容性。

### （2）多模态 AI 房产顾问（Homesee Agent）

系统的智能中枢负责视觉理解与语义推理。通过集成视觉语言模型（如 Qwen-VL），实现基于当前全景画面的视觉问答能力；结合大语言模型（如 DeepSeek）完成多轮对话管理与用户意图识别。当检测到视觉依赖意图标记时，系统可自动触发视觉分析模块，形成语言—视觉融合的闭环交互。

### （3）3D 伴随式交互机器人

系统设计并实现了 3D 卡通风格的智能交互机器人，作为情感化交互载体。机器人具备 Idle、Thinking、Speaking 等多种动画状态，通过状态机进行平滑切换，并结合语音播报与动态气泡反馈，增强系统的临场感与交互友好度。

### （4）LLM 驱动的智能匹配引擎

系统摒弃传统协同过滤算法，构建基于 RAG 与大模型推理的智能推荐机制。通过将用户画像、历史行为与房源结构化数据注入模型上下文，由大模型完成语义推理、排序并生成自然语言推荐理由，实现高解释性的个性化匹配。

## 2. 拟解决的主要问题

（1）针对 AI 缺乏视觉感知的问题，提出“实时截帧 + 视觉语言模型分析”的解决方案，使 AI 能够理解当前房源画面并回答细节问题。

（2）针对跨平台 3D 性能与兼容性问题，设计“小程序原生壳 + Web-View 内嵌 3D 核心”的混合架构，实现性能与体验的平衡。

（3）针对大模型幻觉与信息失真问题，采用系统提示词强约束与结构化数据注入的双重机制，确保房源信息的准确性与可靠性。

（4）针对长链路交互延迟问题，引入流式响应与前瞻性反馈设计，通过动画与气泡文本缓解等待感，提升整体交互体验。

# 三、课题研究方法、设计方案与论文撰写提纲

## 1. 课题研究方法

本研究采用原型驱动的敏捷开发方法，遵循“设计—实现—评估”的迭代流程；同时采用前后端分离的模块化设计思想，通过清晰的接口契约支持多端并行开发。在关键技术难点上，采取问题拆解、技术选型与集成验证相结合的方法完成系统实现。

## 2. 系统设计方案

系统整体采用云端前后端分离架构，划分为交互层、AI 中台、服务层与数据层。核心交互流程围绕“语音提问—意图识别—多模态处理—结果合成—沉浸式反馈”展开，形成完整的 AI 视觉问答闭环。

## 3. 论文撰写提纲

论文共分为六章，依次为绪论、相关理论与关键技术、系统需求分析与总体设计、系统核心模块实现、系统测试与效果评估，以及总结与展望。

# 四、总体安排与进度

课题研究周期共 24 周，依次完成系统原型开发、云端部署、小程序开发、系统优化以及论文撰写与答辩准备等阶段，确保研究工作与工程实现同步推进。

# 五、课题研究准备情况

目前已完成 Three.js、Web Speech API、多模态大模型接口、高德地图 SDK 等关键技术验证，具备云服务器、域名与备案条件，并已搭建完整的全栈开发与部署环境，为课题顺利实施奠定了坚实基础。
