# 毕业设计（论文）开题报告

**项目题目：** 融合大模型交互与 3D 全景预览的智能选房平台设计与实现

---

## 选题的背景与意义

**背景：** 当前，主流线上租赁平台的数字化展示仍普遍停留在“二维图片列表”形式，缺乏对房源空间属性与居住环境的深度还原与理解。用户无法获得身临其境的观感，更难以就视觉内容（如布局、采光、装修细节）进行即时交互与质询。为此，本研究设计并实现了一个融合 **Web 3D沉浸式展示** 与 **多模态大模型智能对话** 的线上看房系统。系统不仅能通过全景技术“呈现”房源环境，更能借助 **视觉-语言融合理解模型**，使AI助手具备“看懂”场景并回答视觉相关问题的能力，从而突破了传统基于文本检索或简单问答的交互局限。

**理论意义：** 本研究探索了多模态大模型在垂直领域（房地产）的落地范式，验证了“视觉感知与自然语言交互闭环”在实际应用中的有效性，为情境感知计算与人机交互研究提供了新的案例参考。所实现的基于 **RAG的智能推荐框架**，为超越传统协同过滤算法、迈向基于深度语义与视觉内容理解的个性化匹配提供了可行路径。

**实践意义：**

*   **交互范式革新：** 成功实现了“Web端全景漫游”与“小程序端轻量化交互”的双端协同，证明了在移动端通过优化架构集成3D渲染与AI服务的可行性，为复杂应用的小程序化开发提供了实践经验。
*   **技术应用深化：** 将大模型从纯粹的文本对话延伸至对三维视觉环境的理解与推理，实现了“即看即问”的沉浸式导览，显著提升了用户体验与信息获取效率。
*   **行业价值：** 为房产租赁行业的数字化转型提供了从“展示”到“理解与交互”的升级方案，通过提升决策效率和看房体验，具有明确的商业应用前景。

## 课题研究的基本内容与拟解决的主要问题

### 一、 课题研究的基本内容

本系统旨在构建一个“可看、可问、可交互”的下一代智能看房平台。其核心由双端交互层、多模态AI中脑与3D数字孪生层协同构成，技术架构与核心功能如下：

#### 1. 双端全景看房交互层
*   **Web端（高性能全景漫游）：** 采用 Vue3+ Three.js技术栈，实现基于射线拾取与相机控制的高性能全景漫游，支持自由视角旋转、缩放与热点导航，为用户提供沉浸式的桌面级看房体验。
*   **小程序端（轻量混合架构）：** 针对移动端性能约束，创新性地采用微信小程序原生框架与 Web-View组件混合架构。通过Web-View无缝承载Web端全景模块，并利用JSBridge实现原生组件与3D页面的双向通信，在保障3D体验的同时，解决了性能与跨平台兼容性难题。

#### 2. 多模态AI房产顾问（Homesee Agent）
这是系统的智能中枢，赋予系统“视觉理解”与“语义推理”能力。
*   **视觉问答模块：** 集成视觉语言大模型（如Qwen-VL）。当用户提出关于画面内容的问题（如“采光如何？”）时，系统自动截取当前全景视图，由该模块进行视觉问答分析，实现“看图说话”。
*   **智能导览与意图理解模块：** 基于深度语言模型（如DeepSeek），具备多轮对话管理与用户意图识别能力。其核心是能解析用户查询，当检测到[NEED_VISION]等视觉依赖标记时，自动调用视觉问答模块，形成“语言-视觉”的闭环交互。

#### 3. 3D 伴随式交互机器人
为增强交互的临场感与友好度，系统设计并实现了一个3D卡通风格智能机器人。它并非简单装饰，而是承担了情感化交互载体的功能：
*   **多状态动画：** 拥有Idle（待机）、Thinking（思考）、Speaking（说话）等骨骼动画状态，通过状态机平滑切换，即时反馈系统处理状态。
*   **语音与气泡交互：** 整合Web Speech API实现语音唤醒与播报，并结合动态对话气泡，提供语音-视觉双通道反馈，有效缓解AI处理延迟带来的等待感。

#### 4. LLM驱动的智能匹配引擎
摒弃传统的协同过滤或标签匹配算法，构建了基于检索增强生成与大模型推理的智能推荐新范式。
*   **方案：** 通过精心设计的提示词工程，将用户画像、历史行为与房源的结构化数据（如价格、户型、位置）注入大模型上下文。
*   **过程：** 系统首先从数据库中进行初步检索，再将相关房源信息与大模型的世界知识结合，由大模型进行推理、排序并生成自然语言推荐理由，实现基于深度语义理解的个性化匹配。

### 二、 拟解决的主要问题

1.  **解决“AI缺乏视觉感知”问题：** 针对传统客服无法理解VR画面的局限，提出了 **“实时截帧 + 视觉语言模型分析”** 的解决方案。当用户问题涉及视觉内容时，系统自动捕获当前全景画面，交由视觉大模型分析，使AI具备了“看懂”房间并回答细节问题的能力。
2.  **解决“跨平台3D性能与兼容性”问题：** 面对High-Poly的3D模型在移动端性能瓶颈，设计了 **“小程序原生壳 + Web-View内嵌3D核心”** 的混合架构。此方案将计算密集的WebGL渲染交由Web-View处理，同时利用小程序原生组件保障外围功能的流畅与一致性，实现了体验与性能的平衡。
3.  **解决“大模型幻觉与信息失真”问题：** 为确保房产信息的绝对准确，采用了 **“系统提示词强约束 + 结构化数据注入”** 的双重保障机制。通过设计严格的System Prompt限定AI的作答范围与角色，并将房源真实数据以结构化格式注入上下文，有效抑制了幻觉，确保了回答的可靠性。
4.  **解决“长链路交互体验迟滞”问题：** 为优化“语音识别→大模型推理→语音合成”长链路带来的延迟体验，运用了 **“流式响应与前瞻性反馈”** 设计。在AI生成文本的同时，机器人即启动Thinking动画与流式气泡文本输出，最后再进行语音合成，将机械等待转化为积极的交互过程，显著提升了用户体验。

## 课题研究方法、设计方案或论文撰写提纲

### 1. 课题研究方法

*   **原型驱动的敏捷开发方法：** 遵循“设计-实现-评估”的迭代循环。首先构建涵盖“3D渲染-AI对话”核心功能的最小可行产品，在本地环境完成闭环验证；随后迭代扩展至云端部署与多端发布，确保系统核心架构的稳健性与可扩展性。
*   **前后端分离的模块化设计方法：** 采用分层与解耦的设计思想。后端基于Spring Boot构建高内聚、低耦合的微服务API；前端采用组件化框架（Vue3）实现交互逻辑。通过定义清晰的接口契约，支持Web端与小程序端的并行开发与灵活集成，并通过Nginx反向代理实现开发与生产环境的一致性配置。
*   **针对复杂问题的技术集成与验证方法：** 面对“多模态融合”与“跨端性能”核心挑战，本研究采用问题拆解-技术选型-集成验证的路径。具体而言：为解“AI视觉理解”问题，集成并评测了Qwen-VL等视觉语言模型；为解“移动端3D性能”问题，设计并验证了“小程序原生壳+Web-View”混合渲染方案的有效性。

### 2. 设计方案

#### 2.1 系统总体架构设计
本系统采用基于云服务的前后端分离分层架构，整体分为交互层、AI中台、服务层与数据层，其核心是以 **多模态AI中台（Homesee Agent）** 为智能中枢的请求调度与响应生成机制。
*   **交互层：** 包括Web端与小程序端，负责3D场景渲染、UI展示及语音/触控指令采集。
*   **AI中台（核心）：** 作为系统的智能核心，封装了视觉理解、意图识别、对话管理与智能推荐服务。它根据用户请求类型，动态调度DeepSeek-V3与Qwen-VL等模型，是“视觉-语言”融合决策的实现层。
*   **服务层：** 基于Spring Boot构建，提供用户、房源、订单等基础业务API，并作为AI中台与数据库之间的桥梁。
*   **数据层：** 由MySQL、Redis及对象存储构成，分别负责结构化数据、缓存及全景图片/3D模型资源的存储。

#### 2.2 核心交互流程设计
一次典型的“AI视觉问答”流程如下：
1.  **请求上传：** 用户在3D场景中语音提问“这个客厅的采光好吗？”，前端完成语音识别后，将文本及当前视角的场景截图上传。
2.  **意图调度：** 请求抵达AI中台，经DeepSeek-V3分析识别出[NEED_VISION]意图标记。
3.  **多模态处理：** AI中台自动调用Qwen-VL服务，对截图进行视觉问答分析，并将结果文本返回。
4.  **响应合成：** DeepSeek-V3整合视觉分析结果与知识库信息，生成最终回复，经语音合成后，驱动3D机器人以语音及动画形式反馈给用户。

    *   第1周：完成项目需求调研，确定前后端分离的技术选型（Spring Boot + Vue3），搭建本地开发环境（JDK, Node.js, MySQL, Redis），完成数据库ER图设计 with 基础建表。
    *   第2周：搭建Spring Boot后端基础框架，实现用户认证（JWT）、房源管理等基础API接口，并完成Swagger接口文档的编写。
    *   第3周：搭建Vue3前端脚手架，集成Element Plus组件库，实现登录注册、房源列表检索等基础页面逻辑，完成与后端的初步联调。
    *   第4周：引入Three.js引擎，研究Web端全景图渲染技术，实现全景场景加载、视角旋转、缩放等基础交互功能。
    *   第5周：集成.glb格式的3D机器人模型，实现模型加载与动画状态机（Idle/Speaking）控制，初步跑通"3D数字人+全景看房"的混合渲染模式。
    *   第6周：对接Web Speech API实现浏览器端的语音识别（STT）与合成（TTS），集成DeepSeek大模型API，实现"语音输入-大模型回答"的本地对话闭环。
