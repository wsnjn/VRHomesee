毕业设计（论文）开题报告

课题名称：基于 Spring Boot 与 Vue3 的沉浸式房屋租赁平台（Homesee）的设计与实现

一、 本课题的背景、目的及研究意义

1.1 课题背景

随着城市化进程的加快和人口流动性的增加，房屋租赁市场规模持续扩大。传统的房屋租赁模式存在信息不对称、看房成本高、虚假房源泛滥等问题。虽然市场上已有贝壳、自如等成熟平台，但它们往往侧重于线下服务流程的数字化，而在“沉浸式体验”和“智能化匹配”方面仍有提升空间。 近年来，Web 3D 技术（如 WebGL、Three.js）和人工智能技术（如 LLM、推荐算法）的飞速发展，为房屋租赁平台提供了新的技术赋能。用户不再满足于静态的图片和文字介绍，而是期望获得更直观、更真实的在线看房体验。

1.2 研究目的

本课题旨在设计并实现一个名为“Homesee”的现代化房屋租赁平台。该平台不局限于传统的信息发布与检索，而是致力于通过技术手段解决“看房难、选房难”的痛点。 具体目的包括：

降低决策成本：通过 VR 全景看房技术，让用户足不出户即可获得身临其境的看房体验，减少无效的线下奔波。

提升匹配效率：利用 AI 智能匹配算法，根据用户的多维度需求（预算、通勤、生活习惯等）精准推荐房源。

构建社区生态：打造租房社区，通过真实评价和社交互动，建立信任机制，改善租赁关系。

1.3 研究意义

理论意义：探索 Web 3D 技术在垂直电商领域的应用模式，以及 AI 大模型在长尾需求匹配中的实践价值，为相关领域的软件工程实践提供参考。

现实意义：

对租客：提供更透明、高效的找房工具，改善租房体验，降低生活成本。

对房东：提供更生动的房源展示手段（VR），提高房源曝光率和出租效率。

对行业：推动房屋租赁行业向“体验化”、“智能化”方向转型升级。

二、 本课题的国内外研究现状

2.1 国外研究现状

在国外，Zillow、Redfin 等房地产平台较早引入了 3D Home Tours 功能，利用计算机视觉技术生成房屋的 3D 模型。Matterport 等公司提供了专业的 3D 扫描硬件和云服务，确立了高端 VR 看房的行业标准。此外，Airbnb 等平台在社区化运营和个性化推荐方面积累了丰富经验，注重用户评价体系和生活方式的匹配。 趋势：国外研究正向着 NeRF (神经辐射场) 和 3D Gaussian Splatting 等 AI 生成 3D 场景技术演进，试图降低 3D 内容的生产成本。

2.2 国内研究现状

国内的贝壳找房、安居客等平台已普及了 VR 看房功能，主要采用全景图拼接技术。自如等长租公寓平台则通过标准化装修和管家服务提升品质。然而，大部分中小平台仍停留在“图片+列表”的传统模式。在智能化方面，虽然已有基于协同过滤的推荐系统，但基于自然语言交互（如 AI 找房助手）的应用尚处于起步阶段。 不足：现有平台往往忽视了租房过程中的社交属性，且 VR 看房内容的生产门槛依然较高，普通房东难以低成本制作高质量 VR 内容。

三、 本课题的研究目标和内容

3.1 研究目标

构建一个集“房源管理、VR 沉浸式看房、AI 智能匹配、社区互动”于一体的综合性房屋租赁平台。系统需具备良好的用户体验、高并发下的稳定性以及易于扩展的架构。

3.2 研究内容

本系统主要包含以下核心模块：

VR 全景看房模块：

基于 Three.js 和 PhotoSphereViewer 实现前端全景漫游。

研究全景图的加载优化策略（如六面体切片加载）以提升 Web 端流畅度。

实现热点交互，支持在不同房间场景间跳转。

AI 智能匹配与助手模块：

集成 AI 大模型接口（如 Gemini/GPT），实现自然语言找房助手（“帮我找一个离地铁近、有阳光的南向卧室”）。

基于用户画像和房源标签的智能推荐算法。

社区与社交模块：

构建“房友圈”，支持发布动态、分享租房经验。

实现即时通讯（IM）功能，支持租客与房东、租客与租客间的实时沟通。

业务管理模块：

房东端：房源发布、合同管理、租客管理、收租提醒。

租客端：在线签约、报修申请、账单支付。

管理端：数据可视化大屏（ECharts）、用户审核、内容监管。

四、 本课题的实行方案

4.1 技术架构

采用前后端分离的 B/S 架构：

前端：Vue 3 + Vite + Pinia + Element Plus。使用 Three.js 实现 3D 场景渲染。

后端：Spring Boot + MyBatis-Plus。提供 RESTful API 接口。

数据库：MySQL 8.0 存储业务数据，Redis 缓存热点数据。

文件存储：自建文件服务器或对象存储（MinIO/OSS），存储全景图和媒体文件。

AI 集成：通过 HTTP Client 调用大模型 API。

4.2 关键技术路线

VR 渲染：利用 Three.js 的 SphereGeometry 或 BoxGeometry 贴图技术，将全景图映射到几何体内部，通过相机视角的旋转模拟观察者视角。

即时通讯：使用 WebSocket 协议实现全双工通信，保证消息的实时送达。

数据可视化：使用 ECharts 绘制动态图表，展示平台运营数据（如房源分布地图、价格趋势）。

五、 预期效果

系统功能完备：成功实现用户注册登录、房源 CRUD、VR 看房、在线签约、社区发帖等核心功能。

交互体验流畅：VR 场景加载速度快，拖拽平滑；AI 助手响应准确。

界面美观现代：前端 UI 采用现代化设计语言，具备良好的视觉效果和响应式布局。

文档规范：产出完整的需求分析文档、系统设计文档、测试报告及毕业论文。
